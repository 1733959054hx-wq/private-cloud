## 〇、语音录入 AI 智能体模块 ✅ 已实现（lqz）

> **定位**：作为键盘打字输入的补充，用户通过语音快速输入文档内容、评论、搜索关键词等文本。

### 0.0.1 语音识别服务选型

- **实际方案**：使用**浏览器内置 Web Speech API（SpeechRecognition 接口）**
- **优势**：零配置、无需任何 API Key、免费无调用次数限制
- **中文识别准确率**：≥95%（标准普通话场景），自动添加标点符号
- **调用方式**：前端直接调用 `SpeechRecognition` / `webkitSpeechRecognition` → 浏览器原生引擎识别 → 返回文字
- **浏览器要求**：Chrome 80+ / Edge 80+（Firefox 和 Safari 不兼容）

### 0.0.2 功能需求

#### 录入入口
- 文档编辑器工具栏提供 🎙️ 语音录入按钮
- 评论输入框、搜索框旁提供麦克风图标快捷入口

#### 操作流程
1. 用户点击麦克风图标，浏览器请求麦克风权限
2. 直接调用浏览器 SpeechRecognition API 开始识别
3. 界面显示录音状态（脉冲动画 + 声波）
4. 用户说完后语音自动结束，浏览器原生引擎转写为文字
5. 识别结果展示在弹窗中，用户可确认/编辑/重录

#### 状态反馈
| 状态 | 提示 |
|------|------|
| 待机 | "点击开始语音录入" |
| 录音中 | 🎙️ 红色脉冲 + 声波动画 + "正在聆听…" |
| 识别中 | ⏳ "正在识别…" |
| 成功 | 显示识别文字，可 ✅确认 / ✏️编辑 / 🔄重录 |
| 失败 | 错误提示 + 一键重试 |

#### 应用场景
- 文档编辑区：语音转文字作为正文内容
- 评论输入框：语音输入评论内容
- 搜索框：语音输入搜索关键词并自动搜索
- AI 问答：语音输入自然语言问题

### 0.0.3 技术约束

- 单次语音识别时长由浏览器自动控制
- 支持语言：中文普通话（zh-CN）
- 浏览器要求：仅 Chrome 80+ / Edge 80+（使用 Web Speech API，Firefox/Safari 不支持）
- 无需后端 API，不依赖任何第三方语音识别服务
- 不支持浏览器的系统会显示"请使用 Chrome 或 Edge 浏览器"提示

## 0. 系统基础与通用能力

### 0.1 用户认证与账号管理
- **企业内部账号体系**：系统不对外开放注册，所有账号由企业管理员在后台统一创建（支持批量导入）
- **登录功能**：登录界面提供用户名/密码输入，无注册入口
- **认证方式**：基于 JWT 的无状态认证，Token 包含用户ID、角色、过期时间
- **密码安全**：密码使用 BCrypt 加密存储，支持修改密码、重置密码（管理员操作）
- **会话管理**：Token 存储在客户端，支持自动刷新；服务端可强制踢出用户（清 Token 缓存）

### 0.2 高性能缓存（Redis）
- **集成 Redis** 作为缓存中间件
- **缓存内容**：用户会话、热点文档元数据、目录树结构、热词统计结果、部门/角色信息
- **缓存策略**：支持过期时间（如会话30分钟）、主动失效（文档更新时清除相关缓存）
- **管理员缓存监控**：在后台提供缓存监控界面，可查看当前Redis中的缓存键数量、内存占用、命中率，支持手动清空指定缓存

- **后端 API**：`GET /api/front/ai/tags/cloud` — 返回全部标签名及出现次数（GROUP BY 聚合统计）；`GET /api/front/ai/tags/file-ids?tagName=xxx` — 按标签名查找关联文件

- **前端组件**：`TagCloud.vue` — `web/src/components/TagCloud.vue`，支持 clickable 模式、v-model 双向绑定激活标签、12 色随机配色、hover 上浮动画、count 驱动字体大小缩放

#### 6.2 安全审计与在线用户管理 (12 分)
- **在线用户管理**
  - 实时监控当前系统在线用户状态
  - 支持管理员强制将违规或异常账户踢出下线
- **行业合规维度的过期文件自动销毁或强制保留策略设置**（定时任务扫描并删除/冻结过期文件）
- **全局操作日志**
  - 记录上传、删除、分享、权限变更等管理操作
- **文件访问日志**（新增）
  - 记录用户访问文件的时间、IP、操作类型（预览、下载、编辑）
  - 支持按用户、文件、时间范围检索与导出

---

### 7. 工作流与审批管理模块 (22 分)

#### 7.1 审批流设计（自建，不集成 Flowable） (15 分)
- **自建审批表**（如 `approval_request`, `approval_step`），实现串行审批（申请人 → 一级审批人 → 二级审批人 或 直接完成）
- **支持简单条件流转**（例如根据文档类型选择不同审批人角色）
- **前端可视化为时间轴或步骤条**

#### 7.2 审批监控与处理 (7 分)
- **全局流转进度可视化**（查看所有审批单状态、当前步骤）
- **管理员强制干预**：可终止、驳回或重新指派审批人

---

### 8. 系统设置与拓展配置模块 (16 分)

#### 8.1 基础字典与组织架构 (8 分)
- **多层级部门与员工角色增删改查**（部门由管理员固定创建，如“技术部”、“财务部”，用户不可自行添加）
- **公司/部门隔离**：管理员可创建不同公司（多租户），每个公司下设置固定部门，避免组织混乱
- **上传文档敏感词过滤词库设置与自动拦截告警配置**（关键词匹配）

#### 8.2 存储与计算引擎集成 (4 分)
- **OCR 引擎**：固定使用一种（百度云 API 或本地 PaddleOCR），提供密钥配置页面
- **存储策略**：支持本地服务器存储与 MinIO 二选一（通过配置文件切换）

#### 8.3 缓存监控与管理 (4 分)
- **Redis 缓存监控**：管理员可在后台查看 Redis 运行状态（内存占用、键数量、命中率）
- **支持手动清除指定缓存**（如清除用户会话缓存、文档目录缓存）
- **提供缓存统计图表**（如近一小时缓存命中率变化）
